## 2. システム概要

### 2.1 システム目的

#### 社内DX業務サポート(年間効果: 1.65億円)
- **粗利改善**: 原価率1.5%改善 → 年間1.65億円増益
- **工程遅延防止**: 指摘是正リードタイム30%削減
- **監督工数削減**: 写真整理時間50%削減(月8時間/人)
- **残業削減**: 現場監督の残業30%削減(月60時間→42時間)
- **原価可視化**: 実績原価確定を月末後2週間→リアルタイムへ

#### 社外向けDXサポート(経審評点+5点目標)
- **発注者評価向上**: 透明性向上による継続受注率+10%
- **維持管理受注獲得**: 竣工データ活用で年間10件目標
- **DXコンサル事業化**: 地方自治体向け新規収益源
- **経審評点向上**: ICT活用で技術評価点+5点

## 3. 機能要件

### 3.1 施工進捗管理機能 (Phase 1)

#### 3.1.1 工程計画作成

**機能概要**
工事案件ごとに工程計画(ガントチャート)を作成し、工期・担当者・作業内容を定義する。

**入力項目**
- 案件ID、案件名
- 工種(土工、コンクリート工、鉄筋工など)
- 作業名、開始日、終了日
- 担当者、協力会社
- 先行作業(依存関係)

**処理内容**
- ガントチャート自動描画
- クリティカルパス算出
- 週次・月次単位での表示切替

**出力**
- ガントチャートPDF出力
- Excelエクスポート(従来フォーマット互換)

**画面イメージ**
- 工程一覧画面(カレンダービュー)
- 工程詳細編集画面
- ガントチャート表示画面

#### 3.1.2 進捗実績登録

**機能概要**
日々の進捗実績を登録し、計画との差異を可視化する。

**入力項目**
- 作業日
- 工種、作業内容
- 進捗率(%)
- 遅延理由(遅延時のみ)
- 写真ID(紐付け)

**処理内容**
- 計画進捗率 vs 実績進捗率の差分算出
- 遅延作業の自動抽出
- 影響工程の自動算出(後続作業への影響)

**出力**
- 進捗率グラフ
- 遅延アラートメール
- 週次進捗レポート自動生成

#### 3.1.3 遅延アラート(公共工事重視)

**機能概要**
進捗遅延を検知し、関係者へ自動通知する。公共工事特有の工期厳守要件に対応。

**トリガー条件**
- 進捗率が計画比▲10%以上乖離
- クリティカルパス上の作業が遅延
- 天候不良による作業中止が3日以上
- **公共工事特有条件**
  - 検査予定日まで2週間を切った作業の進捗率80%未満
  - 債務負担行為案件の年度末1ヶ月前時点で進捗率90%未満
  - 繰越手続き必要案件の早期検知(進捗率から逆算)

**通知先**
- 現場監督(即時)
- 本社工事部門責任者(即時)
- 工事部門長(重要案件のみ)
- 協力会社担当者(該当作業のみ)

**通知方法**
- メール(優先度: 高/中/低で件名に表示)
- システム内通知(ダッシュボード赤色表示)
- (Phase 2以降)Slack/Teams連携
- (緊急時)SMS通知(現場監督の携帯へ)

#### 3.1.4 工程×原価連動

**機能概要**
工程進捗と原価実績を紐付け、工種別の原価進捗を可視化する。

**データ連携**
- 工程進捗率
- 工種別原価予算
- 工種別原価実績(外注費、材料費、労務費)

**分析内容**
- 工種別原価進捗率 = 実際原価 ÷ (予算原価 × 進捗率)
- 原価進捗率 > 1.0 の場合、原価超過アラート

**出力**
- 工種別原価進捗グラフ
- 原価乖離アラート

### 3.2 写真・出来形管理機能 (Phase 1)

#### 3.2.1 写真登録(スマホアプリ)

**機能概要**
現場でスマホから工事写真を登録する。GPS位置情報、工種、撮影日時を自動記録する。

**入力項目**
- 案件選択
- 工種選択(マスタから選択)
- 撮影種別(着手前、施工中、完成)
- 写真(カメラ撮影 or ギャラリー選択)
- 備考(任意)

**自動記録項目**
- GPS座標(緯度・経度)
- 撮影日時
- 撮影者(ログインユーザー)
- 工程自動紐付け(現在進行中の工程と自動紐付け)

**処理内容**
- 画像リサイズ(最大2MB)
- EXIF情報保持
- クラウドストレージへアップロード
- サムネイル自動生成

**制約**
- オフライン時は端末内に一時保存、オンライン復帰時に自動アップロード

#### 3.2.2 写真承認フロー

**機能概要**
登録された写真を工程責任者が確認・承認する。

**ワークフロー**
1. 現場作業員: 写真登録(ステータス=未承認)
2. 現場監督: 写真確認
   - 承認 → ステータス=承認済
   - 差戻 → ステータス=要修正(差戻理由入力)
3. 本社管理者: 最終確認(Phase 2以降)

**承認条件**
- 撮影日時が妥当
- 工種が正しく紐付いている
- 画像品質が十分(手ぶれ、ピンボケ検知機能: Phase 2)

**出力**
- 承認済写真一覧
- 未承認写真一覧(アラート)
- 承認履歴(監査証跡)

#### 3.2.3 写真帳自動生成(公共工事電子納品対応)

**機能概要**
承認済写真を工種・日付順に整理し、写真帳(PDF)を自動生成する。公共工事の電子納品基準に完全準拠。

**レイアウト設定**
- 1ページ2枚 / 4枚 / 6枚(選択可能)
- 写真下部に自動印字
  - 案件名、工種名、撮影日時、GPS座標、撮影者
  - **公共工事必須項目**: 工事番号、施工箇所、測点、出来形寸法

**出力フォーマット**
- **PDF(電子納品対応)**: CALS/EC準拠
- **XMLファイル**: 電子納品要領準拠(PHOTO.XMLなど)
- Excel(従来フォーマット互換)

**納品対応(公共工事対応強化)**
- **国土交通省電子納品基準(CALS/EC)準拠**: 必須
  - デジタル写真管理情報基準
  - 工事完成図書の電子納品等要領
- **自治体独自フォーマット対応**
  - テンプレート登録機能(Phase 1で主要5自治体分を標準搭載)
  - カスタムテンプレート追加機能(Phase 2)
- **ファイル命名規則自動適用**
  - PHOTO_001.JPG形式など、発注者指定の命名規則に自動変換
- **電子納品チェック機能**
  - 国交省「電子納品チェックシステム」との連携(Phase 2)
  - 納品前の自動検証(ファイル形式、命名規則、XMLスキーマ)

#### 3.2.4 GPS位置連携

**機能概要**
写真撮影位置をGISマップ上で表示する。

**表示内容**
- 地図上にピン表示(撮影位置)
- ピンクリックで写真プレビュー
- 工種別に色分け表示

**活用用途**
- 施工範囲の可視化
- 未施工箇所の特定
- 発注者への位置情報提供(Phase 3)

### 3.3 指摘・是正管理機能 (Phase 1)

#### 3.3.1 指摘事項登録

**機能概要**
現場で発見した問題点・改善事項を登録し、関係者と共有する。

**入力項目**
- 案件ID
- 指摘日
- 指摘箇所(工種)
- 指摘内容(テキスト)
- 重要度(高・中・低)
- 対応期限
- 対応担当者(協力会社)
- 写真添付(任意)

**処理内容**
- 指摘番号自動採番
- 対応期限アラート設定
- 関係者へメール自動送信

#### 3.3.2 是正対応登録

**機能概要**
指摘事項への対応状況を登録する。

**入力項目**
- 指摘番号
- 対応日
- 対応内容(テキスト)
- 対応完了写真(必須)
- ステータス(対応中/対応完了/保留)

**承認フロー**
1. 協力会社: 是正対応登録
2. 現場監督: 対応内容確認
   - 承認 → ステータス=完了
   - 再指摘 → ステータス=対応中(コメント追加)

#### 3.3.3 指摘管理ダッシュボード

**機能概要**
全案件の指摘状況を一覧表示する。

**表示項目**
- 案件別未対応件数
- 期限超過件数(赤色アラート)
- 重要度別件数
- 協力会社別対応状況

**フィルタ機能**
- 案件名、工種、担当者、ステータス

**出力**
- 指摘一覧Excel出力
- 週次指摘サマリレポート(PDF)

### 3.4 原価モニタリング機能 (Phase 1)

#### 3.4.1 予算登録

**機能概要**
案件ごとに原価予算を登録する。

**入力項目**
- 案件ID
- 工種別予算
  - 材料費
  - 外注費
  - 労務費
  - 経費
- 粗利目標(%)

**処理内容**
- 工種別予算積算
- 全社予算集計

#### 3.4.2 実績原価登録

**機能概要**
発生した原価を登録する。

**入力方法**
- 手動入力(Web管理画面)
- CSVインポート(既存会計システムから)
- API連携(外部システムから自動取得: Phase 2)

**入力項目**
- 案件ID
- 工種
- 発生日
- 費目(材料費/外注費/労務費/経費)
- 金額
- 業者名
- 伝票番号(証憑管理用)

#### 3.4.3 原価乖離アラート

**機能概要**
予算と実績の乖離を検知し、アラートを発報する。

**アラート条件**
- 工種別原価進捗率 > 1.1(予算比10%超過)
- 粗利率が目標比▲3%以上悪化
- 月次原価が予算ペースを超過

**通知先**
- 現場監督
- 本社工事部門責任者
- 経営層(重大案件のみ)

**通知内容**
- 乖離金額
- 乖離要因(分析機能: Phase 2)
- 対策提案(AI分析: Phase 3)

#### 3.4.4 粗利リアルタイム可視化

**機能概要**
案件ごとの粗利をリアルタイムで可視化する。

**表示内容**
- 予算粗利 vs 実績粗利
- 粗利率推移グラフ
- 完成時見込粗利(進捗率から推定)

**集計単位**
- 案件別
- 工事部門別
- 全社合計

**更新頻度**
- 日次(原価登録時に即時反映)

## 7. 制約事項

### 7.1 技術的制約(IT部門5名で運用可能な技術選定)

**開発言語・フレームワーク(選定基準)**
- **バックエンド**: Python/Django または Node.js/Express(候補)
  - 選定基準: IT部門がサポート可能、クラウドマネージドサービス豊富
- **フロントエンド**: React または Vue.js(候補)
  - 選定基準: 採用情報が多い、コミュニティサポート豊富
- **モバイル**: PWA優先(ネイティブアプリは Phase 3以降検討)
  - 理由: ストア申請不要、IT部門の負荷軽減

**インフラ(マネージドサービス優先)**
- **クラウド**: AWS または Azure(候補)
  - 選定基準: マネージドサービス豊富、IT部門がインフラ管理不要
  - 必須サービス: マネージドDB、オブジェクトストレージ、自動バックアップ、監視サービス
- **データベース**: PostgreSQL(マネージドDB: AWS RDS / Azure Database)
  - 理由: オープンソース、GISデータ対応(PostGIS)、レプリケーション標準機能
- **ストレージ**: S3(AWS)またはBlob Storage(Azure)
  - 容量: 初期1TB、自動拡張
- **CDN**: CloudFront(AWS)またはAzure CDN
  - 写真配信高速化

**IT部門負荷軽減のための制約**
- **サーバー管理不要**: フルマネージドサービス利用
- **自動スケーリング**: 負荷に応じた自動サーバー増減
- **自動バックアップ**: 日次自動バックアップ、復元テスト自動化
- **監視自動化**: クラウド標準監視サービス利用(CloudWatch等)

**ブラウザサポート**
- IE11非対応(サポート終了のため)
- 対応ブラウザ: Chrome, Edge, Safari, Firefox(各最新版+1世代前)

**データ保持期間**
- 案件データ: 竣工後5年間
