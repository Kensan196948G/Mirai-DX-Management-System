## 1. エグゼクティブサマリ

### プロジェクト概要

本プロジェクトは、公共事業中心の建設土木会社(従業員550名)が、社内の建設工事管理業務をデジタル化・効率化するための統合プラットフォーム「Construction Digital Control Platform (CDCP)」を構築・運用するものです。

**プロジェクトの二大目的**:
1. **社内DX**: 現場監督の負荷軽減、原価率改善、工期管理効率化
2. **社外DX**: 発注者向けサービス、維持管理事業化、DXコンサル事業化

## 3. プロジェクトスコープ

### 3.1 スコープ内容

#### Phase 1 (0-4ヶ月): パイロット運用 - 基盤構築

**対象現場**: 2現場(中規模公共工事)
**対象ユーザー**: 現場監督10名、協力会社20名、本社管理部門5名(計35名)

**実装機能**:
- 施工進捗管理(工程計画、進捗実績登録、遅延アラート、工程×原価連動)
- 写真・出来形管理(スマホ撮影・登録、GPS自動記録、工程自動紐付け、承認フロー、電子納品対応)
- 指摘・是正管理(指摘登録、是正対応、期限管理、ダッシュボード)
- 原価モニタリング(予算登録、実績登録、乖離アラート、粗利可視化)
- 基本ダッシュボード

**成果物**:
- システムシステム設計書、API仕様書
- 操作マニュアル、操作動画(10本)
- ユーザー研修資料
- 運用手順書

#### Phase 2 (4-9ヶ月): 全社展開 - スケールと連携

**対象**: 全社30-40現場
**対象ユーザー**: 現場監督80名、協力会社300名、本社20名、工事部門長8名(計408名)

**追加機能**:
- 同時接続418名への対応(性能強化、クラウド自動スケーリング)
- 会計システムCSV自動連携(日次バッチ)
- 帳票テンプレート拡張(自治体別テンプレート10種)
- ワークフローカスタマイズ機能
- 多要素認証(MFA)対応
- ヘルプデスク機能強化(FAQ充実、チャットボット検討)

**成果物**:
- 性能最適化レポート
- 帳票テンプレート10種
- ヘルプデスク運用マニュアル

#### Phase 3 (9-18ヶ月): 社外展開 - ビジネス拡大

**対象**: 発注者ポータル、維持管理サービス
**対象ユーザー**: 全社ユーザー+発注者ユーザー(約100-150名)

**実装機能**:
- 発注者向け進捗可視化ポータル(リアルタイム進捗、写真証跡、GPS連携)
- 竣工データパッケージ自動生成(デジタル台帳、GIS連携データ、CALS/EC準拠)
- 維持管理サービス機能(点検管理、損傷管理、報告書自動生成)
- 地方自治体向けDX支援機能
- BI連携(経営分析ダッシュボード)
- 外部API公開(協力会社向け、JV向け)

**成果物**:
- 発注者ポータル実装
- 維持管理サービス機能
- DXコンサル標準サービス仕様書

### 3.2 スコープ外

以下は本プロジェクトスコープ外です(将来検討対象):

- AI画像品質チェック(Phase 2で見送り、高コスト)
- 予測AI(原価予測、工期予測はPhase 3後期以降)
- ネイティブモバイルアプリ(PWA優先)
- SaaS事業化(Phase 3後期以降検討)
- 外国人対応(英語翻訳機能はPhase 3以降)

### 3.3 スコープ変更管理

スコープ外の要件追加・変更については、別途「変更管理手順書」に従い、承認プロセスを経て対応します。

### 7.2 リスク対応計画

**R1: 現場のデジタル抵抗感**
- **発生時**: パイロット現場での利用促進不足
- **対応**: ベテラン監督(デジタル先進者)をチャンピオンに指定、モチベーション管理
- **予防**: UI/UXテストに現場監督を参加

**R2: IT部門の運用負荷過多**
- **発生時**: ヘルプデスク問い合わせが計画を超過
- **対応**: 外部ヘルプデスク契約、24時間シフト体制移行
- **予防**: 運用負荷予測ツール導入、週単位での工数監視

**R3: 開発遅延**
- **発生時**: スケジュール遅延で1ヶ月以上の遅延
- **対応**: ベンダーリソース追加、機能削減検討、Phase 2への繰越
- **予防**: 2週間スプリント、毎週進捗レビュー

**R4-R8**: 各リスク監視指標を月次レビューで確認、早期警告時に対応計画実行

### 7.3 リスク監視・報告

- **月次**: プロジェクト会議でリスク状況報告
- **四半期**: 経営会議でリスク・機会報告
- **リスク担当**: プロジェクトマネージャー

## 9. 制約条件

### 9.1 技術的制約

- **IT部門5名で運用可能な設計必須**: クラウドマネージドサービス優先、サーバー管理不要
- **オンプレミス廃止**: Phase 1からクラウド前提、既存サーバーは段階廃止
- **多言語対応**: Phase 1-2は日本語のみ、Phase 3で英語対応検討
- **ブラウザサポート**: IE11は非対応(Chrome, Edge, Safari各最新版)

### 9.2 運用制約

- **稼働時間**: 24/365利用可能(定期メンテナンス月1回除く)
- **定期メンテナンス**: 月1回深夜(2-4時)のみ
- **ユーザーサポート**: Phase 1-2は平日営業時間、Phase 3で24時間対応検討

### 9.3 法的制約

本プロジェクトは以下の法令・基準への準拠が必須:
- **建設業法**: 施工体制台帳、下請負契約要件
- **個人情報保護法**: 個人情報取得・利用・管理
- **電子帳簿保存法**: 電子データの真正性・保存要件
- **CALS/EC**: 公共工事電子納品基準(必須)
- **ISO 27001**: 情報セキュリティ(Phase 2目標)
- **Pマーク**: 個人情報保護(Phase 3目標)

### 9.4 予算・期間制約

- **Phase 1予算**: 3,500万円で固定(追加予算は原則不可)

## 12. 成功の重要要因(CSF)

### 12.1 Critical Success Factors

1. **経営層のコミット**: DX推進を経営戦略の筆頭に位置づけ、資金・人員・時間を確保
   - 指標: 経営会議での月次レビュー実施率100%

2. **現場の巻き込み**: ベテラン監督をキーマンとして早期参画、意見を最大限反映
   - 指標: パイロット現場の満足度≧80%

3. **スモールスタート**: Phase 1は2現場に限定、確実な成果を出す
   - 指標: Phase 1成功基準達成率100%

4. **IT部門の負荷管理**: 外部リソース活用で持続可能な運用体制構築
   - 指標: IT部門の運用工数≦10時間/人/週

5. **KPI可視化**: 効果を定量的に示し、全社展開の推進力とする
   - 指標: 月次KPI達成率≧80%
