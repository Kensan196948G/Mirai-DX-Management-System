# Phase3 開発計画書 (Development Plan)

## 1. Phase3 概要

### 1.1 フェーズ目的
- 顧客ポータル構築による差別化サービス提供
- 電子納品対応による業務効率化
- 外部連携による生態系構築

### 1.2 対象範囲
- **対象拠点**: 40拠点（継続）
- **対象ユーザー**: 500+名（社内418名 + 顧客・協力会社）
- **対象顧客**: 主要顧客50社
- **開発期間**: 9ヶ月（10-18ヶ月目）

### 1.3 投資規模
- **開発費用**: 5,700万円
  - 顧客ポータル開発: 3,500万円
  - 電子納品対応: 1,500万円
  - 外部連携: 500万円
  - セキュリティ強化: 200万円

## 4. 主要機能仕様

### 4.1 顧客ポータル機能

#### 4.1.1 案件進捗閲覧
- **機能**: 顧客が自社案件の進捗をリアルタイム確認
- **要件**:
  - 案件一覧表示（自社案件のみ）
  - Ganttチャート閲覧
  - 進捗率表示
  - 工程写真閲覧（共有設定された写真のみ）
  - 週次進捗レポートダウンロード
- **アクセス制御**:
  - マルチテナント（顧客別データ分離）
  - 案件別アクセス権限
  - IPアドレス制限（オプション）

#### 4.1.2 帳票ダウンロード
- **機能**: 各種帳票のダウンロード
- **帳票種類**:
  - 工事写真帳（PDF）
  - 出来形管理図表（PDF）
  - 品質管理図表（PDF）
  - 工程表（PDF/Excel）
  - 原価明細（承認された範囲のみ）
- **要件**:
  - 最新版の自動反映
  - バージョン管理（過去版も閲覧可）
  - ダウンロード履歴記録

#### 4.1.3 コミュニケーション機能
- **機能**: 顧客との双方向コミュニケーション
- **要件**:
  - 問い合わせチケット管理
  - 写真へのコメント・指摘
  - 是正依頼ワークフロー
  - 通知機能（メール + ポータル内通知）

### 4.2 電子納品対応

#### 4.2.1 CALS/EC準拠
- **機能**: 国土交通省CALS/EC基準準拠の電子納品データ生成
- **対応基準**:
  - 工事完成図書の電子納品要領（H28年3月版）
  - 電子納品運用ガイドライン
  - デジタル写真管理情報基準
- **生成データ**:
  - 電子納品フォルダ構成
  - INDEX_C.XML（工事管理ファイル）
  - DRAWING.XML（図面管理ファイル）
  - PHOTO.XML（写真管理ファイル）
  - SXF図面ファイル
  - 工事写真（JPEG + Exif）

#### 4.2.2 電子納品チェック
- **機能**: 電子納品データの妥当性チェック
- **チェック項目**:
  - フォルダ構成チェック
  - XMLスキーマ検証
  - ファイル命名規則チェック
  - 画像解像度チェック
  - メタデータ整合性チェック
- **対応**: 電子納品検査システム連携（オプション）

#### 4.2.3 帳票自動生成
- **工事写真帳**:
  - レイアウトテンプレート（4枚/頁、6枚/頁）
  - 工種・工程別自動分類
  - 撮影位置図自動生成（GPS利用）
  - 黒板情報OCR（将来拡張）
- **出来形・品質管理帳票**:
  - 測定データ自動集計
  - 規格値との比較グラフ
  - 度数分布図自動生成

### 4.3 外部連携API

#### 4.3.1 API公開
- **公開範囲**: 顧客・協力会社向けAPI
- **認証**: OAuth 2.0 + API Key
- **提供API**:
  - 案件情報取得
  - 工程情報取得
  - 写真一覧取得
  - 帳票ダウンロード
- **レート制限**: 顧客あたり1,000リクエスト/時

#### 4.3.2 外部システム連携（オプション）
- **会計システム連携**: 原価データ自動連携
- **勤怠システム連携**: 労務費自動計上
- **CAD連携**: 図面データインポート

## 5. 非機能要件

### 5.1 性能要件

| 項目 | 目標値 | 備考 |
|------|--------|------|
| **同時接続ユーザー** | 1,000名 | 社内500名 + 顧客500名 |
| **顧客ポータル応答** | 2秒以内 | 初期表示 |
| **帳票生成** | 30秒以内 | 工事写真帳100頁 |
| **電子納品データ生成** | 5分以内 | 1案件分 |
| **API応答** | 1秒以内 | 95パーセンタイル |

### 5.2 セキュリティ要件

| 項目 | 要件 | 実現方法 |
|------|------|---------|
| **マルチテナント分離** | 顧客間データ完全分離 | Row-Level Security (RLS) |
| **顧客認証** | 外部IdP連携 | SAML 2.0 / OIDC |
| **API認証** | OAuth 2.0 | Auth0 API |
| **通信暗号化** | TLS 1.3 | 必須 |
| **侵入検知** | WAF | AWS WAF |
| **脆弱性診断** | 外部診断 | リリース前 + 年1回 |

### 5.3 可用性要件

| 項目 | 目標値 | Phase2との差分 |
|------|--------|---------------|
| **稼働率** | 99.9%（月間停止43分以内） | 0.4%向上 |
| **RTO（目標復旧時間）** | 2時間 | 2時間短縮 |
| **RPO（目標復旧時点）** | 1時間 | データロス最小化 |

## 6. 顧客展開計画

### 6.1 展開戦略

**段階的顧客オンボーディング**

| Wave | 対象顧客数 | 顧客属性 | 展開期間 |
|------|-----------|---------|---------|
| **Wave1** | 5社 | パイロット顧客（協力的） | Week 67-68 |
| **Wave2** | 20社 | 主要顧客（高取引額） | Week 68-69 |
| **Wave3** | 25社 | その他顧客 | Week 69-70 |

### 6.2 顧客オンボーディングプロセス

1. **事前準備（1週間前）**
   - 顧客マスタ登録
   - アカウント発行
   - 初期設定（案件割当、権限設定）
2. **キックオフ（1時間）**
   - ポータル説明
   - デモンストレーション
   - Q&A
3. **トレーニング（2時間）**
   - 操作研修
   - 実案件での試用
   - FAQ提供
4. **フォローアップ（1ヶ月）**
   - 週1回の利用状況確認
   - 問い合わせ対応
   - 改善要望収集

### 6.3 顧客サポート体制

| 項目 | 内容 |
|------|------|
| **専用窓口** | 顧客専用ヘルプデスク |
| **対応時間** | 平日9-18時 |
| **対応方法** | メール、電話、チャット |
| **SLA** | 1営業日以内に初回回答 |
